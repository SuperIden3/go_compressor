RLE - Run Length Encoding

Run Length Encoding, abbreviated as RLE, is an encoding algorithm that can be used to compress data. It is a lossless data compression algorithm that replaces repeated sequences data with a shorter length.

This algorithm is a very common algorithm in computer science and is used in many applications, such as in image compression, video compression, and data compression.

Basic Idea

RLE replaces continuous sequences of the same value with a pair:

  1. A number that indicates how many times the value occurs (we’ll call this the run length)
  2. The value itself

For example, the sequence "AAAAAA" would be replaced with "6A".

Implementation

However, in this implementation, the run length is not actually restricted to the character values of '1' to '9' (values 49 to 57). The run length is a single binary character, meaning it can range from 1 to 255.

Instead, the sequence "AAAAAA" would then be replaced with "<STX>A".

This binary characteristic of the run length enables itself to communicate a wider range of values while also being shorter.

Managing Overflow

In this implementation, because the maximum value is 255 for the run length, it will write the current pair with the run length of 255 (either "ÿ" or a no-break space) and then subtracts the run length by the maximum value. This is done until the run length is less than the maximum value.

(Assuming 255 = ÿ) For a sequence like character "C" repeated 520 times, the output will be "ÿCÿC C", where the space has a character code of 10.

The output is as it is because (same character code value assumptions): ÿ + ÿ + S = 255 + 255 + 10 = 510 + 10 = 520

Limitations

  • Input & Output

    RLE is great for files that have continuous sequences of the same value, but absolutely sucks at random data. In the worst case, RLE will double the size of the input.

    Take "ABABABAB". It will return "<SOH>A<SOH>B<SOH>A<SOH>B<SOH>A<SOH>B<SOH>A<SOH>B". This output doubles the size of the input.

    This makes RLE a rare choice for data compression, but it is still useful for files that are mostly continuous sequences of the same value.
